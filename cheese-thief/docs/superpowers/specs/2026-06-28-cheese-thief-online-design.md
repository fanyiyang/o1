# 奶酪大盗 在线版 · 设计文档

- 日期：2026-06-28
- 状态：已确认，开始实现

## 1. 目标

把桌游《奶酪大盗 (Cheese Thief)》做成一个**4 人在线**的网页小游戏，托管在 GitHub Pages 上，朋友之间用浏览器（手机/电脑）即可联机游玩。说话/讨论靠玩家自己的语音工具（微信、Discord、面对面），App 只负责需要保密的游戏机制。

## 2. 游戏规则（4 人基础局）

- **身份**：3 张睡鼠（村民）+ 1 张奶酪大盗，洗匀后秘密发牌，每人一张。4 人局没有共犯。
- **骰子**：每人秘密摇一颗 1–6 的骰子，只有本人能看到自己的点数。
- **夜晚（闭眼）**：睡鼠睡觉不行动；奶酪大盗睁眼把桌子中央的奶酪偷走藏进自己骰盅。
- **白天（讨论）**：玩家开语音自由辩论、撒谎、推理谁偷了奶酪。
- **投票**：所有人同时指认一名怀疑对象，得票最多者翻开身份牌。
- **胜负**：
  - 投出的是大盗 → **睡鼠阵营赢**。
  - 投出的是睡鼠 → **大盗赢**。
- **平票规则（让骰子有意义，忠于原版“算 7”）**：按“有效点数”决出被翻牌者——拿到奶酪的大盗有效点数算 **7（最大）**，睡鼠为其骰子点数。并列最高票者中，有效点数最高者翻牌。因此大盗只要并列最高票就会被翻出（“奶酪出卖了你”）；若并列者全是睡鼠，则点数高者翻牌。极端情况下点数也并列，则按玩家 id 排序确定（保证结果确定、可测试）。

## 3. 架构

- **技术栈**：纯静态 `HTML/CSS/JS` 单页，无构建步骤；ES Modules；PeerJS 通过 CDN 引入。
- **联网**：WebRTC 玩家直连（PeerJS），**房主浏览器当“裁判”**。
  - 房主 `new Peer(roomCode)`，房间号即房主的 peer id（如 `CHS-7K2`）。
  - 其余玩家 `peer.connect(roomCode)` 直连房主。
  - 信令走 PeerJS 免费公共服务器；**零后端、零账号**。
- **保密 / 防作弊**：裁判（房主）只把每个人**自己**的身份和骰子私下发给本人；其他人的身份/骰子在结算前**绝不**经网络发到客户端，因此中途开开发者工具也看不到别人的牌。唯一信任假设：房主不去翻自己浏览器内存作弊（朋友局可接受）。

### 3.1 模块划分

- `js/game.js` — **纯逻辑**，无 DOM / 无网络。发牌、摇骰、计票、平票决议、胜负判定。可被 Node 直接 import 做单测。
- `js/net.js` — PeerJS 封装。房主端（建房、接受连接、广播 + 私发）、玩家端（连接、收发）。定义消息协议。
- `js/app.js` — 把 UI、网络、逻辑粘起来；驱动状态机。
- `index.html` / `style.css` — 单页多屏，手机端友好，奶酪主题。
- `test/game.test.mjs` — `node:test` 跑的纯逻辑单测。

### 3.2 状态机（阶段）

`lobby → dealing → night → day → voting → result`

1. **lobby**：房主建房得房间号；玩家输房间号 + 昵称加入；满 4 人房主点「开始」。
2. **dealing**：裁判发身份 + 摇骰，私发给每人；每人看到自己的卡牌 + 点数。
3. **night**：睡鼠看“你睡着了”；大盗看“偷奶酪”并点击确认。
4. **day**：展示玩家列表，玩家开语音讨论；房主点「开始投票」。
5. **voting**：每人同时选一个目标；票齐后裁判结算。
6. **result**：翻牌 + 揭晓所有身份与骰子；显示胜负；房主可「再来一局」。

### 3.3 消息协议（JSON over PeerJS DataConnection）

玩家 → 房主：
- `{type:'join', name}`
- `{type:'night-done'}`（大盗偷完奶酪 / 睡鼠确认）
- `{type:'vote', target}`

房主 → 单个玩家（私密）：
- `{type:'role', role, die}`（仅发给本人）

房主 → 全体（广播，仅公开信息）：
- `{type:'players', list:[{id,name}]}`
- `{type:'phase', phase}`
- `{type:'vote-tally', counts}`（投票结束后）
- `{type:'result', eliminated, winner, reveal:[{id,name,role,die}]}`

## 4. 错误处理 / v1 局限

- 连接失败 → 提示检查房间号并重试。
- 房主掉线 → 通知全体“本局结束”，回到首页。
- 玩家中途掉线 → 显示掉线提示；v1 不做重连（刷新会掉线，重开房间即可）。
- 必须正好 4 人才能开始。
- 不做账号、不存档、不做排行榜。

## 5. 测试

- **纯逻辑**：`game.js` 全部用 `node:test` 覆盖（发牌恒有 1 大盗、计票、平票决议、胜负），可注入随机数保证确定性。
- **联网/界面**：本地起静态服务，开 4 个浏览器标签页（1 房主 + 3 加入）手动联调一局完整流程。
